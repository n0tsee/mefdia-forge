# Сборка Windows-версии (через GitHub Actions)

Windows-установщик (`.exe` и `.msi`) **нельзя собрать на Mac** — его собирает
GitHub в облаке на Windows-раннере. Настроено уже всё: workflow лежит в
`.github/workflows/build.yml`. Нужно только залить проект в GitHub и запустить.

## Что делает сборка автоматически

- Поднимает Windows-раннер, ставит Node + Rust.
- **Сама скачивает** `ffmpeg.exe`/`ffprobe.exe` (полная сборка с gyan.dev) и
  кладёт их как sidecar — коммитить 80-мегабайтные бинарники не нужно.
- Собирает `.exe` (установщик NSIS) и `.msi`.
- Кодировщик видео на Windows выберется сам: NVIDIA NVENC → Intel Quick Sync →
  AMD AMF, а если GPU/драйвер не поддерживает — откат на CPU (libx264).
- Заодно собирает `.dmg` для macOS на том же прогоне.

## Шаги (один раз)

### 1. Установить Git и авторизоваться (если ещё нет)

```bash
brew install gh
gh auth login        # выбрать GitHub.com → HTTPS → войти в браузере
```

### 2. Создать репозиторий и залить проект

Из папки проекта (`mediaforge`):

```bash
cd "/Users/Apple/Desktop/разработка/mediaforge"
git init
git add .
git commit -m "MediaForge"
gh repo create mediaforge --private --source=. --push
```

> `--private` — репозиторий приватный (код никто не увидит). Сборка в Actions
> для приватных репозиториев тоже бесплатна в рамках месячного лимита.

### 3. Запустить сборку

Сборка стартует сама после `push`. Смотреть прогресс:

```bash
gh run watch
```

или на сайте: вкладка **Actions** в репозитории.

### 4. Скачать готовый установщик

Когда прогон позеленеет (~5–10 минут):

```bash
gh run download          # скачает mediaforge-windows/ и mediaforge-macos/
```

или на сайте: **Actions → последний запуск → Artifacts** внизу страницы:
- `mediaforge-windows` — внутри `MediaForge_0.1.0_x64-setup.exe` и `.msi`
- `mediaforge-macos` — `.dmg`

Файл `MediaForge_..._x64-setup.exe` — это и есть то, что скидываешь людям.

## Как обновлять версию

Подними номер в `package.json` и `src-tauri/tauri.conf.json` (`version`),
запушь. Хочешь официальный Release с прикреплёнными файлами — поставь тег:

```bash
git tag v0.1.0
git push --tags
```

Появится черновик Release со всеми установщиками — останется нажать «Publish».

## Что увидит Windows-пользователь при установке

Приложение **не подписано** сертификатом (как и macOS-версия). При запуске
`.exe` Windows покажет синее окно **«Система Windows защитила ваш компьютер»**
(SmartScreen). Это ожидаемо для неподписанных приложений. Нужно нажать:

**«Подробнее» → «Выполнить в любом случае»**

После установки приложение запускается обычным двойным кликом. Никаких команд
в терминале, в отличие от macOS, не требуется.

> Убрать это предупреждение полностью можно только платной подписью кода
> (сертификат Authenticode / EV, от отдельного поставщика). Для раздачи между
> знакомыми это не обязательно.

## Требования к машине пользователя

- Windows 10/11 (64-бит).
- **WebView2** — установщик поставит его сам, если отсутствует (в новых
  Windows он уже встроен).
